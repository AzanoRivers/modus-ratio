### T4.2 — Bloque `@theme` con tokens Tailwind y breakpoints

El bloque `@theme` convierte las variables CSS en tokens de Tailwind. La diferencia clave respecto a `azanolabs-web`: el breakpoint `xl` es **600px** (no 1280px) y se elimina `2xl`.

**Sintaxis a verificar con F0**: `azanolabs-web` usa `@theme inline`. Confirmar en `F0-hallazgos.md` si Tailwind 4.x en Astro usa `@theme` o `@theme inline`. Si F0 no ha resuelto esto aún, usar la variante que funcione al compilar.

```css
@theme {
  /* Colores → clases bg-background, text-foreground, bg-neon-blue, etc. */
  --color-background:    var(--background);
  --color-foreground:    var(--foreground);
  --color-az-grid-bg:    var(--az-grid-bg);
  --color-az-grid-line:  var(--az-grid-line);
  --color-neon-pink:     var(--neon-pink);
  --color-neon-blue:     var(--neon-blue);
  --color-neon-green:    var(--neon-green);

  /* Fuentes → clases font-orbitron, font-audiowide, font-signature, font-mono */
  --font-orbitron:   var(--font-orbitron);
  --font-audiowide:  var(--font-audiowide);
  --font-signature:  var(--font-signature);
  --font-mono:       var(--font-mono);

  /* Breakpoints de Modus Ratio (sobreescriben los defaults de Tailwind) */
  --breakpoint-sm:  270px;
  --breakpoint-md:  360px;
  --breakpoint-lg:  440px;
  --breakpoint-xl:  600px;
  /* Sin 2xl — Modus Ratio es una sola vista, no necesita ese rango */
}
```

**Criterio de aceptación**: `xl:hidden` oculta un elemento en DevTools a 600px (no a 1280px); `md:text-red-500` activa a 360px.

### T4.3 — Instalación y carga de fuentes

En `azanolabs-web` las fuentes se cargan con `next/font/google`. En Astro usamos `@fontsource/*` para servirlas **localmente** (sin petición a Google Fonts en producción).

**Instalar**:
```
pnpm add @fontsource/orbitron
pnpm add @fontsource/audiowide
pnpm add @fontsource/space-grotesk
```

**Importar en `globals.css`** (antes del bloque `:root`):
```css
@import '@fontsource/orbitron';
@import '@fontsource/audiowide';
@import '@fontsource/space-grotesk';
@import 'tailwindcss';
```

El orden importa: las fuentes antes de `tailwindcss` para que Tailwind las reconozca.

**Variantes a importar**: por defecto `@fontsource` importa el peso 400. Orbitron necesita también el 700 (bold para los títulos neón):
```css
@import '@fontsource/orbitron/400.css';
@import '@fontsource/orbitron/700.css';
@import '@fontsource/audiowide/400.css';
@import '@fontsource/space-grotesk/400.css';
@import '@fontsource/space-grotesk/500.css';
```

**Criterio de aceptación**: en DevTools > Network, al cargar la página demo no hay ninguna petición a `fonts.googleapis.com` ni `fonts.gstatic.com`. Los archivos `.woff2` se sirven desde el dominio local.

### T4.5 — Estilos globales de base (HTML, body y scrollbar global)

Portar los estilos globales que aplican a todos los elementos de la página.

**Contenido a incluir**:

```css
html,
body {
  margin: 0;
  padding: 0;
}

/* Scrollbar global neón — aplica a todos los elementos */
* {
  scrollbar-color: rgba(11, 210, 255, 0.4) rgba(4, 21, 40, 0.6);
  scrollbar-width: thin;
}

::-webkit-scrollbar        { width: 6px; height: 6px; }
::-webkit-scrollbar-track  { background: rgba(4, 21, 40, 0.6); }
::-webkit-scrollbar-thumb  { background: rgba(11, 210, 255, 0.35); border-radius: 3px; }
::-webkit-scrollbar-thumb:hover {
  background: rgba(11, 210, 255, 0.65);
  box-shadow: 0 0 6px rgba(11, 210, 255, 0.5);
}
::-webkit-scrollbar-corner { background: transparent; }

body {
  color: var(--foreground);
  font-family: var(--font-mono); /* fuente de cuerpo base */
}

/* Dark mode (la app siempre es dark, pero se respeta la preferencia del sistema) */
@media (prefers-color-scheme: dark) {
  :root {
    --background: #041528;
    --foreground: #ededed;
  }
}
```

**Nota sobre `font-family` del `body`**: en azanolabs-web el body usa `font-family: Arial, Helvetica, sans-serif` como fallback. En Modus Ratio usamos `var(--font-mono)` como fuente base del cuerpo (los títulos y elementos específicos sobreescriben con `font-orbitron` o `font-audiowide`). Si F0 o el diseño final indican lo contrario, se ajusta en F7.

**Criterio de aceptación**: el scrollbar en cualquier elemento con scroll muestra el color azul neón; `body` no tiene margin ni padding por defecto.

### T4.6 — Ajustes responsive del grid (media queries)

El grid `.bg-az-grid` cambia de tamaño en breakpoints mayores para mantener la proporción visual. Se adaptan los valores de `azanolabs-web` a los breakpoints de Modus Ratio.

**Tabla de tamaños**:

| Breakpoint MR | px | Grid size | Motivo |
|---|---|---|---|
| Default (< sm) | < 270px | 25px | Pantallas muy pequeñas |
| `md` | ≥ 360px | 35px | Igual que la referencia (360px) |
| `lg` | ≥ 440px | 40px | Igual que la referencia (440px) |
| `xl` | ≥ 600px | 45px | Nuevo: xl pasó de 1280px a 600px; escala ligeramente |

```css
@media (min-width: 360px) { .bg-az-grid { background-size: 35px 35px; } }
@media (min-width: 440px) { .bg-az-grid { background-size: 40px 40px; } }
@media (min-width: 600px) { .bg-az-grid { background-size: 45px 45px; } }
```

**Criterio de aceptación**: en la página demo, al arrastrar el viewport de 270px a 600px se aprecia visualmente que las celdas del grid se agrandan progresivamente.

### T4.7 — Archivo `constants.css`

Constantes reutilizables que no son tokens de diseño (tokens = `:root` + `@theme`) sino valores derivados que se usarán en múltiples componentes para cálculos o referencias. Se importa desde `globals.css`.

**Contenido inicial**:

```css
:root {
  /* Grid */
  --grid-size-default: 25px;
  --grid-size-md: 35px;
  --grid-size-lg: 40px;
  --grid-size-xl: 45px;

  /* Bordes neón */
  --neon-border-width: 2px;

  /* Scrollbar global */
  --scrollbar-width-global: 6px;
  --scrollbar-width-component: 8px;

  /* Intensidades de glow para text-shadow */
  --glow-spread-near: 4px;
  --glow-spread-mid: 8px;
  --glow-spread-far: 12px;
  --glow-spread-outer: 16px;

  /* Transición base de estado (hover, focus, etc.) */
  --transition-base: 200ms ease;

  /* Semáforo — umbrales del ratio (se usan en F16) */
  --ratio-green-min: 70;   /* ≥ 70 → verde */
  --ratio-yellow-min: 40;  /* ≥ 40 → amarillo */
                           /* < 40 → rojo */
}
```

**Por qué los umbrales del semáforo aquí**: son constantes numéricas que el CSS (CSS counter, custom properties) y el TSX (comparaciones) van a leer. Centralizarlas evita que estén hardcodeadas en dos lugares distintos. En TSX se leerán como constantes TS en un archivo separado (`src/lib/constants.ts`) que espeja estos valores — esa constante TS se crea en F16.

**Criterio de aceptación**: `constants.css` existe e importado en `globals.css`. Todos los valores tienen nombre semántico.

### T4.8 — `BaseLayout.astro`

El layout base que replican todas las páginas de Modus Ratio como si fueran "internas" de `azanolabs-web`. Acepta props para el `<head>` y una slot para el contenido.

**Estructura que debe producir en el HTML final**:

```
<html lang="{locale}" class="overflow-x-hidden">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>{title} — Modus Ratio</title>
    <meta name="description" content="{description}" />
    <!-- OG tags básicos -->
    <link rel="icon" ... />
  </head>
  <body class="antialiased min-h-screen w-full flex flex-col bg-az-grid">
    <main class="flex-1 flex flex-col">
      <slot />             ← contenido de cada página
    </main>
    <footer class="shrink-0 py-3 flex justify-center">
      <!-- Firma mínima: "Modus Ratio · AzanoLabs" -->
    </footer>
  </body>
</html>
```

**Props del layout**:
```
interface Props {
  title: string        ← texto de la pestaña (sin el sufijo "— Modus Ratio")
  description: string  ← meta description
  locale: Locale       ← para el atributo lang del <html>
}
```

**Notas de implementación**:
- El `<PageTransitionWrapper>` (equivalente de azanolabs-web) no existe aún. Se deja un comentario `<!-- TODO F5: PageTransitionWrapper -->` en el lugar donde irá.
- `globals.css` se importa aquí como `import '../styles/globals.css'` (o con el alias `@/styles/globals.css`).
- La firma del footer es texto estático (hardcodeado aquí como excepción justificada: es un `<!-- -->` de marca que no necesita traducción, similar a copyright). **Alternativa**: si el equipo decide traducirlo, agregar clave i18n en F3 antes de usar. Por defecto: `Modus Ratio · AzanoLabs`.
- La directiva de hidratación para el futuro wrapper de transición se decide en F5.

**Criterio de aceptación**: todas las páginas de Astro que usen `BaseLayout` producen HTML con `<html lang="es">` o `<html lang="en">`, el body con `bg-az-grid`, y el `<title>` correcto. `pnpm build` sin errores.

## Riesgos

| Riesgo | Mitigación |
|---|---|
| `@theme` vs `@theme inline` en Tailwind 4 + Astro: sintaxis diferente | Confirmar en `F0-hallazgos.md` (T0.7). Si no está resuelto, probar ambas variantes y usar la que compila |
| `@fontsource` de Audiowide puede tener nombre de paquete distinto (`@fontsource/audiowide` podría no existir) | Verificar en npmjs.com antes de `pnpm add`. Alternativa: `@font-face` manual con archivo `.woff2` descargado de Google Fonts |
| Componentes futuros portados de `azanolabs-web` usan `xl:` con estilos pensados para 1280px | Cada portado de componente en F6+ revisará y ajustará las clases `xl:` — está documentado en la decisión #1 del fundational plan |
| `prefers-reduced-motion` global (F5) puede colisionar con el reset de F4 | El `prefers-reduced-motion` de F4 aplica solo a transiciones de hover/focus genéricas; F5 agrega el suyo propio para `.page-enter`. No colisionan si están en selectores diferentes |
