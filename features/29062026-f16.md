## Desviaciones y adiciones respecto al plan original

- **Bug de hidratación corregido**: el ejemplo de T16.4 importaba `DIMENSION_KEYS` desde el barrel `@/lib`, que también re-exporta módulos server-only (`env`, `redis`, `r2`, `openai`, `minimax`). Como `env.ts` valida variables de entorno al cargar el módulo, hidratar `ResultsPanel` en el cliente reventaba buscando `REDIS_URL`. Se corrigió importando `DIMENSION_KEYS` directo de `@/lib/analysisTypes` (ver memoria `project-lib-barrel-hydration`).
- **Variables CSS corregidas**: el plan usaba `--color-neon-yellow` (no existe) y `--color-neon-pink` para el estado rojo (semánticamente inconsistente). Se usaron en su lugar `--color-success`/`--color-warning`/`--color-error` (y sus `-glow`), que ya estaban en `constants.css` preparados exactamente para el semáforo de F16. También se sustituyeron `--color-text-muted`, `--color-text`, `--color-surface-raised`, `--color-border`, `--font-display`, `--font-body` (ninguno existe) por los tokens reales del proyecto (`rgba(var(--ch-fg), N)`, `var(--foreground)`, `var(--font-orbitron)`, `var(--font-mono)`, etc.).
- **`Skeleton` real no acepta `width`/`height`/`rounded`**: el plan asumía esa API. Se implementó con `variant` + clases `.results-panel__skel-*` con selector compuesto (`.skeleton.results-panel__skel-*`) para ganarle en especificidad a los tamaños que traen las variantes de `Skeleton.css` (evita el bug del círculo del semáforo estirándose por el `min-height` de `variant="card"`).
- **Imports por barrel**: se corrigieron `Badge`/`Button`/`Skeleton` (`@/components/atoms`), `DimensionBar` (`@/components/molecules`) y `Translations` (`@/i18n`) para usar los barrels de nivel en vez de las rutas internas del plan.
- **Adiciones pedidas por el usuario tras revisar la demo** (fuera del alcance original de T16.1-T16.6, documentadas aquí para trazabilidad):
  - Layout de escritorio en 2 columnas a partir de `2xl` (1200px), con `NeonBreakLine` (extendido con prop `orientation: 'horizontal' | 'vertical'`) como divisor entre columnas.
  - Tooltip de información por dimensión: átomo nuevo `Tooltip` (`src/components/atoms/Tooltip/`) con efecto de aparición `ctTooltipGlitchIn` (mismo lenguaje que `ctGlitchIn` de F5), más el namespace `t.analysis.dimensionDescriptions` (5 descripciones cortas, ES/EN).
  - Botón "Analizar otro outfit" con el mismo efecto de borde centellante que "Analizar outfit": se extrajo el CSS de `HomeForm.css` a la utilidad compartida `.btn-beam` en `globals.css` en vez de duplicarlo.
- La página de demo temporal `src/pages/results-demo.astro` y `src/components/_demo/ResultsDemoControls.tsx` siguen en el repo a la fecha de este cierre; se eliminan en F17 según la nota de ejecución original de este sub-plan.

## Objetivo

Transformar el JSON de `AnalysisResults` en una pantalla visualmente rica, gamificada y legible. El usuario debe poder entender su puntuación de un vistazo (semáforo + ratio global), profundizar en cada dimensión (barras), saber qué sale mal (warnings), qué sale bien (highlight) y qué puede hacer (recomendaciones accionables). Todo el texto viene de i18n — ningún string en pantalla proviene directamente del modelo.

---

## Decisiones ya tomadas

| Decisión | Valor |
|---|---|
| Datos de entrada | `AnalysisResults` desde el store (`results`) |
| Semáforo — icono | verde: ✓ (check), amarillo: ⚠ (advertencia), rojo: ✗ (prohibido) — mapeados desde clase CSS, no emoji directos |
| Colores de barra por score | 0–39: var rojo neón, 40–69: var amarillo neón, 70–100: var verde neón — calculados desde el valor, no desde `semaphore` global |
| Aviso de estilo personal | Texto fijo del diccionario — siempre visible, no condicional |
| Botón de re-análisis | Siempre visible en la vista de resultados |
| Skeletons | Mismos bloques que los reales — misma estructura, mismas dimensiones, placeholder animado mientras `phase === 'analyzing'` |
| Hidratación | `client:idle` — los resultados aparecen después de la interacción inicial |
| CSS modules | No — `.css` de componente con clases globales (convención del proyecto) |
| Barras | Animadas al montar (fill desde 0 hasta el valor) — usando CSS transitions, no JavaScript |

## Cierre — Definition of Done

- Dado un `AnalysisResults` de ejemplo con valores conocidos, la UI muestra todos los bloques correctamente.
- Las barras se animan al montar (0 → valor) con CSS transition.
- El color de cada barra refleja el score de esa dimensión (no el semáforo global).
- El semáforo global muestra el icono y texto correctos para verde, amarillo y rojo.
- Con `phase === 'analyzing'` (y `results === null`), se muestran los skeletons de F6 en lugar de cada bloque.
- Ningún texto en pantalla está hardcodeado — todo viene de `t`.
- Sin `style={{}}` salvo CSS custom properties con valores dinámicos.
- La UI es responsive en los 4 breakpoints (270 / 360 / 440 / 600px).
- `pnpm build` sin errores TypeScript.

### T16.2 — `DimensionBar` (molecule)

Crear `src/components/molecules/DimensionBar/`.

**Responsabilidad**: renderizar una sola dimensión — su nombre localizado, su score (0–100) como número y como barra de progreso coloreada por el score.

**`DimensionBar.tsx`**:

```tsx
import './DimensionBar.css'

interface DimensionBarProps {
  label:  string   // texto ya localizado (viene de t.analysis.dimensions[key])
  score:  number   // 0–100, entero
}

export function DimensionBar({ label, score }: DimensionBarProps) {
  const colorClass =
    score >= 70 ? 'dimension-bar--green'
    : score >= 40 ? 'dimension-bar--yellow'
    : 'dimension-bar--red'

  return (
    <div className="dimension-bar">
      <div className="dimension-bar__header">
        <span className="dimension-bar__label">{label}</span>
        <span className="dimension-bar__score">{score}</span>
      </div>
      <div className="dimension-bar__track">
        <div
          className={`dimension-bar__fill ${colorClass}`}
          style={{ '--score': score } as React.CSSProperties}
        />
      </div>
    </div>
  )
}
```

**Notas de implementación**:

- `style={{ '--score': score }}`: CSS custom property con valor calculado en JS — excepción permitida. El porcentaje de la barra (`calc(var(--score) * 1%)`) se calcula en CSS.
- `colorClass` se determina por el score de la dimensión individual, no por el `semaphore` global del resultado. Un outfit puede tener `semaphore: 'red'` globalmente pero una dimensión en verde.
- Los nombres de variables CSS (`--color-neon-green`, `--color-neon-yellow`, `--color-neon-pink`, `--color-surface-raised`, etc.) deben verificarse contra `globals.css` de F4. El Implementador confirma los nombres exactos.
- `--color-neon-yellow`: verificar si existe en F4. Si no existe con ese nombre, usar el equivalente amarillo/ámbar definido en el design system.

**`index.ts`**:
```ts
export { DimensionBar } from './DimensionBar'
```

**Criterio de aceptación**: score 75 → clase `dimension-bar--green`, barra al 75% de ancho con animación. Score 55 → amarillo. Score 25 → rojo. `prefers-reduced-motion` desactiva la animación.

### T16.4 — `ResultsPanel.tsx`

Crear `src/components/organisms/ResultsPanel/ResultsPanel.tsx`.

```tsx
import './ResultsPanel.css'
import { useAppStore }   from '@/store'
import { DimensionBar }  from '@/components/molecules/DimensionBar'
import { Badge }         from '@/components/atoms/Badge'
import { Skeleton }      from '@/components/atoms/Skeleton'
import { DIMENSION_KEYS } from '@/lib/analysisTypes'
import type { Translations } from '@/i18n/types'

interface ResultsPanelProps {
  t: Pick<Translations, 'analysis' | 'recommendations' | 'results'>
  onReanalyze: () => void
}

const SEMAPHORE_SYMBOL: Record<'green' | 'yellow' | 'red', string> = {
  green:  '✓',
  yellow: '!',
  red:    '✗',
}

export function ResultsPanel({ t, onReanalyze }: ResultsPanelProps) {
  const phase   = useAppStore((s) => s.phase)
  const results = useAppStore((s) => s.results)

  if (phase === 'analyzing' || !results) {
    return <ResultsPanelSkeleton />
  }

  const { dimensions, globalRatio, semaphore, warnings, highlight, recommendations } = results

  return (
    <div className="results-panel">

      {/* Hero: semáforo + ratio global */}
      <div className="results-panel__hero">
        <div className={`results-panel__semaphore results-panel__semaphore--${semaphore}`}>
          {SEMAPHORE_SYMBOL[semaphore]}
        </div>
        <span className="results-panel__ratio-label">{t.results.globalRatio}</span>
        <span className="results-panel__ratio-value">{globalRatio}</span>
        <span className="results-panel__semaphore-text">{t.analysis.semaphore[semaphore]}</span>
      </div>

      {/* Dimensiones */}
      <div className="results-panel__section">
        <h3 className="results-panel__section-title">{t.results.dimensionsTitle}</h3>
        {DIMENSION_KEYS.map((key) => (
          <DimensionBar
            key={key}
            label={t.analysis.dimensions[key]}
            score={dimensions[key]}
          />
        ))}
      </div>

      {/* Warnings — solo si hay alguna */}
      {warnings.length > 0 && (
        <div className="results-panel__section">
          <h3 className="results-panel__section-title">{t.results.warningsTitle}</h3>
          <div className="results-panel__warnings">
            {warnings.map((key) => (
              <Badge key={key} variant="warning">
                {t.analysis.warnings[key]}
              </Badge>
            ))}
          </div>
        </div>
      )}

      {/* Highlight — solo si existe */}
      {highlight && (
        <div className="results-panel__section">
          <h3 className="results-panel__section-title">{t.results.highlightTitle}</h3>
          <p className="results-panel__highlight">{t.analysis.highlights[highlight]}</p>
        </div>
      )}

      {/* Recomendaciones */}
      {recommendations.length > 0 && (
        <div className="results-panel__section">
          <h3 className="results-panel__section-title">{t.results.recsTitle}</h3>
          <ol className="results-panel__recs">
            {recommendations.map((key) => (
              <li key={key} className="results-panel__rec">
                <span className="results-panel__rec-text">{t.recommendations[key]}</span>
              </li>
            ))}
          </ol>
        </div>
      )}

      {/* Disclaimer estilo personal */}
      <p className="results-panel__disclaimer">{t.results.styleDisclaimer}</p>

      {/* Botón re-análisis */}
      <Button onClick={onReanalyze}>{t.results.reanalyzeButton}</Button>

    </div>
  )
}
```

**Notas de implementación**:

- `SEMAPHORE_SYMBOL`: mapeo de semáforo a símbolo textual ASCII. No usa emoji para evitar inconsistencias de renderizado entre plataformas. Si el Implementador quiere usar iconos SVG (de F6 o de la librería del proyecto), puede sustituir este mapeo.
- `DIMENSION_KEYS` importado de `@/lib/analysisTypes`: garantiza que el orden de las barras es siempre el mismo y que están todas. No se itera sobre `Object.keys(dimensions)` porque el orden no estaría garantizado.
- `onReanalyze` es un callback prop: el componente no ejecuta lógica de flujo directamente — F17 lo conecta al store y a la máquina de estados.
- El `Button` viene de F6 (`src/components/atoms/Button`). El `Badge` con `variant="warning"` viene de F6. El `Skeleton` viene de F6.
- `Pick<Translations, 'analysis' | 'recommendations' | 'results'>`: el componente solo recibe los 3 namespaces que necesita, no el objeto `Translations` completo.

**Por qué el skeleton está inline y no en un archivo separado**: el skeleton de `ResultsPanel` es un único estado transitorio de este componente. No tiene lógica propia ni reutilización fuera de aquí. Separarlo en un archivo solo agrega un archivo sin beneficio.

## Riesgos

| Riesgo | Mitigación |
|---|---|
| Variables CSS de F4 con nombres distintos (`--color-neon-green` vs `--neon-green`, etc.) | El Implementador abre `globals.css` de F4 y verifica los nombres exactos antes de codificarlos en `DimensionBar.css` y `ResultsPanel.css` |
| `color-mix()` no soportado en targets mínimos | Verificar en F0-hallazgos.md los targets del proyecto. Fallback: `rgba(r, g, b, 0.15)` con los valores RGB de los colores neón |
| `Skeleton` de F6 no acepta las props (`width`, `height`, `rounded`) que usamos aquí | Verificar la API de `Skeleton` en F6 antes de llamarlo. Si tiene props distintas, adaptar la llamada en `ResultsPanelSkeleton` |
| `Badge` de F6 no tiene `variant="warning"` | Verificar en F6 los variantes disponibles (`success`, `warning`, `error`, `info`). Si falta, F6 debe actualizarse antes de F16 |
| La animación de barras no funciona con CSS custom property en `@keyframes` en algunos navegadores | `ctBarFill` usa `from { width: 0 }` y `to { width: calc(var(--score) * 1%) }`. El `to` referencia la custom property en tiempo de animación — verificar en Safari iOS. Fallback: quitar `@keyframes` y usar `transition: width 600ms ease-out` sobre la propiedad `width` directamente (requiere que el elemento empiece con `width: 0` y transite al valor) |
| `recommendations` puede tener 0 elementos | La sección se renderiza condicionalmente con `recommendations.length > 0`. No aparece bloque vacío |
| `warnings` puede tener 0 elementos | Idem — sección condicional |
